# REL-003: Submit place-bet transactions through RPC abstraction

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

Submit signed transactions to Solana devnet RPC. RPC abstraction layer wraps connection for future multi-provider failover. Handle RPC errors, timeouts, and retries.

## Acceptance Criteria

- [ ] Transactions submit to devnet successfully
- [ ] RPC abstraction wraps Solana connection
- [ ] RPC errors handled with retry logic
- [ ] Timeout handling for unresponsive RPC
- [ ] Retry count configurable
