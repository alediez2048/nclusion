# HIS-001: Build user bet history endpoint

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

GET /api/history — returns user's bets grouped by status (open, pending, settled). Paginated. Supports caching headers for offline mobile access.

## Acceptance Criteria

- [ ] Endpoint returns bets grouped by status
- [ ] Pagination works (cursor or offset)
- [ ] Cache-control headers set for mobile caching
- [ ] Empty state handled
- [ ] Filters by status supported
