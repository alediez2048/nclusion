# REL-005: Expose bet lifecycle status updates to betting service

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

Relay notifies betting service of status changes (submitted, confirmed, failed). Betting service updates bet record and internal state machine. Status flows to mobile via next API request.

## Acceptance Criteria

- [ ] Relay emits status updates to betting service
- [ ] Betting service updates bet record on each transition
- [ ] Internal state machine transitions validated
- [ ] Mobile receives updated status on next request
- [ ] Failed transactions trigger fund release
