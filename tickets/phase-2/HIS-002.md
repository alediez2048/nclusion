# HIS-002: Build bet receipt endpoint

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** Medium

## Description

GET /api/history/:bet_id — full receipt. Fields: match, market, accepted odds, stake, potential payout, status, timestamps. Advanced fields: tx_signature, program_account_ref, settlement_signature.

## Acceptance Criteria

- [ ] Receipt returns all standard fields
- [ ] Advanced fields populated after chain confirmation
- [ ] Settlement fields populated after settlement
- [ ] 404 for unknown bet_id
- [ ] Receipt data consistent with on-chain state
