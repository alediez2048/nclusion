# BET-004: Accept bet intent with idempotency key

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

POST /api/betting/intents — accepts bet intent from mobile client. Client generates idempotency key. Server deduplicates on key match. Returns intent_id and initial status.

## Acceptance Criteria

- [ ] Endpoint accepts bet intent with required fields
- [ ] Idempotency key prevents duplicate processing
- [ ] Duplicate request returns same response (not error)
- [ ] intent_id generated and returned
- [ ] Status set to intent_created
