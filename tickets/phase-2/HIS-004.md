# HIS-004: Store recent bet history locally for offline access

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** Medium

## Description

Cache bet history in op-sqlite on the mobile app. Serve from cache when offline. Sync with server on reconnect. Bounded retention for old entries.

## Acceptance Criteria

- [ ] Bet history cached in op-sqlite after each fetch
- [ ] Cached history viewable when offline
- [ ] Syncs with server on reconnect
- [ ] Old entries pruned based on retention policy
- [ ] Cache freshness indicated to user
