# HIS-003: Show user-facing statuses in mobile app

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

Map internal states to user-facing statuses (Draft, Queued, Processing, Confirmed, etc.) in the mobile app. Display status on history list items and bet detail screens.

## Acceptance Criteria

- [ ] All internal states map to correct user-facing status
- [ ] Status displayed on history list items
- [ ] Status displayed on bet detail screen
- [ ] Status updates on screen refresh
- [ ] No raw internal state shown to user
