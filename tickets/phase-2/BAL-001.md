# BAL-001: Implement available, reserved, and pending settlement balance model

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

Drizzle schema for balance_views table. Reserve funds on bet acceptance, release on cancellation/failure. Move to pending_settlement after chain confirmation. Credit winnings after settlement. All transitions must be atomic.

## Acceptance Criteria

- [ ] Balance view schema created with Drizzle
- [ ] Reserve funds atomically on bet acceptance
- [ ] Release funds on cancellation or failure
- [ ] Transition to pending_settlement on chain confirmation
- [ ] Credit winnings after settlement
- [ ] All transitions tested through full lifecycle
