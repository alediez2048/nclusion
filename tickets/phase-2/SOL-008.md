# SOL-008: Enforce duplicate prevention and status constraints

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

Reject bet if bet_id PDA already exists (Anchor init constraint). Enforce valid status transitions in all instructions (can't settle an already-settled bet, can't place on a closed market).

## Acceptance Criteria

- [ ] Duplicate bet_id rejected by Anchor init constraint
- [ ] Invalid status transitions return specific errors
- [ ] All valid transitions tested
- [ ] All invalid transitions tested
- [ ] Error messages descriptive

- **Blocks:** QA-004
- **Blocked by:** SOL-005, SOL-006
