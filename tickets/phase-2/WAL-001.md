# WAL-001: Create managed wallet on first user setup

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

Generate Solana keypair server-side on user registration. Store encrypted private key with custody boundary. Map wallet public key to user record. User never sees seed phrase.

## Acceptance Criteria

- [ ] Keypair generated on registration
- [ ] Private key encrypted at rest
- [ ] Public key stored in user record
- [ ] No seed phrase or private key exposed to client
- [ ] Wallet mapping retrievable for relay service

- **Blocks:** WAL-002, BAL-002, REL-002
- **Blocked by:** API-003, SOL-004
