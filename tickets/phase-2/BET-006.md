# BET-006: Reserve funds in balance service

**Phase:** 2 — Core Betting Loop
**Status:** ⏳ Pending
**Priority:** High

## Description

On validated bet intent, atomically reserve stake amount from available balance. Release if bet fails or is cancelled. Use PostgreSQL transaction for atomicity. Prevent double-reserve.

## Acceptance Criteria

- [ ] Funds reserved atomically on valid intent
- [ ] Available balance decreases by stake amount
- [ ] Reserved balance increases by stake amount
- [ ] Released on bet failure or cancellation
- [ ] Double-reserve prevented
- [ ] Database transaction used for atomicity

## Dependencies

- **Blocks:** REL-001 (funds must be reserved before relay)
- **Blocked by:** BET-005, BAL-001
