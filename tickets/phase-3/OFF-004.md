# OFF-004: Revalidate queued intents on reconnect

**Phase:** 3 — Demo Hardening
**Status:** ⏳ Pending
**Priority:** High

## Description

When connectivity returns, check pending drafts/intents. Revalidate odds and market status before submission. If odds moved outside tolerance, prompt user to reconfirm.

## Acceptance Criteria

- [ ] Pending intents checked on reconnect
- [ ] Odds revalidated against current server odds
- [ ] Market availability rechecked
- [ ] Out-of-tolerance odds trigger reconfirmation prompt
- [ ] Expired intents auto-cancelled with explanation
